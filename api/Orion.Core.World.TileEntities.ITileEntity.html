Remarks

Implementations must be thread-safe.

Many Terraria tile-based objects are tile entities and have common properties such as position. Tile entities are annotatable, allowing consumers to easily attach custom state to them.

There are three types of tile entities:

  1. Tile entities which are abstract: they cannot affect the world.
  2. Tile entities which are concrete but not active: the entity does not exist in the world.
  3. Tile entities which are concrete and active: the entity exists in the world.

These types can be differentiated using the IsConcrete(ITileEntity) extension method and the IsActive property.
